# 右轮 PI 自动调参固件设计

## 目标

将现有按键启动、左右轮可选、带说明文本的调参程序，改为仅针对右轮的自动循环阶跃测试固件。设备上电后立即启动右轮，持续输出机器可解析的 CSV 数据，并接收大模型发送的 PI 参数命令。OLED 实时显示当前运行和控制参数。

## 运行流程

- 完成 `SYSCFG_DL_init()`、编码器、OLED 和速度控制器初始化后，立即启动 10 ms 控制节拍。
- 右轮目标速度按 `300 -> 500 -> 700 -> 500 -> 300 mm/s` 循环，每档保持 2 s。
- 循环不自动停止；左轮目标和输出始终保持为 0。
- 控制计算周期保持 10 ms，串口遥测周期保持 20 ms。
- 不再扫描 KEY1~KEY4，也不再通过按键启动或停止调参。

## 串口发送协议

RESV1_UART 仅发送以下八列 CSV 数据，不发送表头、启动信息、状态信息、确认、错误或结束说明：

```text
timestamp_ms,setpoint,input,pwm,error,p,i,d
```

每行实际数据示例：

```text
254400,300.00,57.00,700.00,243.00,20.000,2.000,0.000
```

字段单位沿用当前工程：时间为 ms，速度为 mm/s；P/I/D 为当前控制器增益。速度、PWM 和误差保留两位小数，增益保留三位小数，`d` 恒为 0。

`timestamp_ms` 为上电后的 10 ms 节拍累计时间，32 位无符号整数自然回绕。

## 串口接收协议

支持 Arduino 示例采用的单行命令：

```text
SET P:20 I:2 D:0
SET P:20.50 I:1.75 D:0
```

- P 和 I 最多解析两位小数，并转换为现有的 `x100` 定点参数。
- 合法范围保持为 `P=0.00~100.00`、`I=0.00~20.00`。
- 当前控制算法仍为 PI，因此 D 必须为 0；非零 D、格式错误、溢出及越界命令均静默忽略。
- 命令以 LF 或 CRLF 结束；接收过程非阻塞。
- 合法参数在运行中立即生效，同时清空右轮积分和当前控制输出，防止继承旧参数的积分状态。
- MCU 不回显命令，也不发送参数更新确认；终端中的蓝色命令由上位机自身显示。

## OLED 显示

OLED 每 500 ms 更新一次，四行固定显示：

```text
SP:+0300 V:+057
PWM:+0700
P:020.00
I:002.00 D:0
```

每行不超过 16 个字符。P/I 显示两位小数，D 固定显示 0。OLED 只承担本地观察，不参与控制或串口协议。

## 文件职责

- `app/pi_tuner.c/.h`：右轮循环阶跃状态、纯 CSV 遥测、`SET P/I/D` 非阻塞解析。
- `app/main.c`：删除按键依赖，上电自动启动，调度控制、串口接收和 OLED 刷新。
- `control/wheel_speed.c/.h`：继续实现 10 ms PI；提供右轮运行时增益更新与诊断数据。
- `bsp/debug_uart.c/.h`：复用 RESV1_UART 字节收发和整数输出，不引入阻塞式字符串读取。
- `bsp/oled.c/.h`：复用现有显示函数；如需要，补充定点小数显示接口。
- `tests/pi_tuner_host_test.c`：验证自动启动、循环阶跃、仅右轮输出、纯 CSV、合法/非法 SET 命令。

## 安全和异常处理

- PWM 继续限制在 `-700~+700`。
- 左轮每个控制周期都保持目标 0、输出 0。
- 串口接收缓冲区溢出时丢弃整行，下一行恢复解析。
- 参数改变时复位 PI 状态，避免瞬间沿用旧积分；电机不会因为命令错误而停止或改变参数。
- 本测试固件没有按键急停。烧录和首次测试时应架空车轮，必要时断开电机电源实现硬件急停。

## 提示词精简范围

`PI大模型调参提示词.md` 只保留三类信息：

1. 调参偏好：响应快、超调小、每次保守调整，输出 `SET P:... I:... D:0`。
2. 不能接受：持续 PWM 饱和、明显持续振荡、积分持续堆积、方向异常或数据不足时继续加参。
3. 已知对象特性：右轮、10 ms 控制、20 ms 遥测、265 count/rev、48 mm 轮径、约 56.9 mm/s 速度量化、PWM 限幅 700、循环目标序列。

## 验证标准

- 主机测试先失败后通过，覆盖协议和状态行为。
- SysConfig 静态检查无新增错误。
- CCS 工程编译包含 `app/pi_tuner.c`、`app/main.c` 和 `control/wheel_speed.c`。
- 实机串口上电后第一条输出就是八字段数值 CSV，整个运行过程中没有 `#`、表头、`OK` 或 `ERR`。
- 发送合法 SET 命令后 OLED 参数改变，后续 CSV 的 P/I 增益值反映新参数；非法命令不改变参数。
- 右轮连续循环阶跃，左轮始终不转。
