# MSPM0G3507 电赛小车完整方案

本文档对应当前 CCS 工程、`empty.syscfg` 和三份数据手册，目标是建立一个可以在竞赛中快速复用的双轮差速小车代码库。

## 1. 当前工程结论

当前工程使用 MSPM0G3507、LQFP-64、MSPM0 SDK 2.10.00.04、SysConfig 1.26.2，CPUCLK 为 32 MHz。

本次检查对 `empty.syscfg` 做了一个必要修改：

- `ENCODER1.interrupts = ["CC0_DN"]`；
- `ENCODER2.interrupts = ["CC0_DN"]`。

这样两路 A 相捕获才会产生捕获事件中断。其余关键配置如下：

| 功能 | 外设/引脚 | 当前用途 |
|---|---|---|
| 右轮 PWM | TIMA1 CCP0 / PB0 | TB6612 PWMA |
| 左轮 PWM | TIMA1 CCP1 / PA16 | TB6612 PWMB |
| 右轮方向 | PA2、PB21 | AIN1、AIN2 |
| 左轮方向 | PB19、PB17 | BIN1、BIN2 |
| 右轮编码器 A/Q | TIMG0 PA23、PB22 | A 相捕获、Q 方向 |
| 左轮编码器 A/Q | TIMG6 PB2、PB13 | A 相捕获、Q 方向 |
| 控制周期 | TIMG7 | 4 MHz、10 ms、ZERO 中断 |
| 循迹输入 | PB18、PB1、PB10、PB11、PB14 | OUT1~OUT5，全部输入 |
| ICM-20948 | I2C0：PA28 SDA、PA31 SCL | 运动传感器 |

TB6612 的 `STBY` 已经硬件接到 3.3 V，因此软件停车必须通过 PWM 和方向输入完成，软件不能再使用 STBY 做故障关断。

## 2. 硬件模块职责与使用方式

### 2.1 MSPM0G3507

MSPM0G3507 负责：

- 产生两路 32 kHz PWM；
- 设置 TB6612 的四个方向输入；
- 捕获两个编码器 A 相上升沿；
- 读取两个 D 触发器 Q 方向信号；
- 读取五路循迹数字输入；
- 通过 I2C 读取 ICM-20948；
- 在 10 ms 控制节拍中运行速度环和循迹控制。

SysConfig 是引脚、时钟、外设和中断的唯一配置源。不要手工修改 `Debug/ti_msp_dl_config.c` 或 `Debug/ti_msp_dl_config.h`。

### 2.2 TB6612FNG

TB6612 负责驱动两个直流减速电机：

- `VCC` 接 3.3 V 逻辑电源；
- `VM` 接 9 V 电机电源；
- 右电机接 AO1/AO2，左电机接 BO1/BO2；
- PWMA/PWMB 使用 TIMA1 两路 PWM；
- AIN1/AIN2、BIN1/BIN2 决定旋转方向；
- STBY 固定为 3.3 V。

电机堵转电流已确认低于 1 A，可以继续使用 TB6612，但仍应测试双电机同时启动、堵转和急停时的芯片温升。VM 侧和 VCC 侧都应靠近驱动器放置 10 uF 与 0.1 uF 去耦电容，电机电源回路使用短而宽的走线。

建议统一使用有符号 PWM 接口：

```c
motor_set_signed(right_permille, left_permille);
```

输入范围为 `-1000~1000`。符号负责方向，绝对值负责占空比。函数内部必须先设置方向，再更新 PWM；占空比为 0 时进入停止状态。

TB6612 的 STBY 固定为高电平，所以停车策略为：

1. 将左右 PWM 比较值设置为 0；
2. 方向输入设置为安全组合；
3. 必要时使用短刹车模式；
4. 故障状态下持续保持 PWM 为 0。

### 2.3 霍尔编码器与 74HC74

当前硬件连接为：编码器 A 相接 MSPM0 捕获输入，编码器 B 相接 74HC74 的 D，编码器 A 相接 CLK，74HC74 的 Q 接方向 GPIO。

该结构的含义是：A 相上升沿到来时，74HC74 锁存 B 电平，Q 表示该边沿对应的方向。Q 不是计数脉冲，计数脉冲来自 A 相捕获事件。

两路捕获当前均为：

- 4 MHz 捕获时钟；
- 10 ms 计时周期；
- A 相上升沿；
- `CC0_DN` 捕获中断。

捕获 ISR 只完成三件事：读取并清除捕获事件、读取对应 Q 电平、更新有符号位置计数。不要在编码器 ISR 中做浮点运算、I2C 通信或 PID 计算。

编码器供电为 3.3 V，若输出确实是 0~3.3 V 推挽脉冲，可以直接连接 MSPM0。74HC74 的 `/PRE`、`/CLR` 必须接到非有效电平，芯片电源和 MSPM0 共地。

### 2.4 五路循迹模块

OUT1~OUT5 从右到左排列。根据已知机械尺寸，默认位置为：

```text
OUT1   OUT2   OUT3   OUT4   OUT5
 +43    +15     0     -15    -43 mm
```

模块输出高电平表示黑线，低电平表示白底。由于实际感光窗口宽度未知，不能假设小车居中时一定是中间三路高电平，必须在实物上采集所有输入组合。

循迹模块驱动层只负责读取五个 GPIO 并返回 bit mask；黑线坐标计算和停车策略放在控制层。

### 2.5 ICM-20948

ICM-20948 通过 I2C0 连接，第一阶段只使用：

- WHO_AM_I 检测；
- 加速度计原始数据；
- 陀螺仪原始数据；
- 温度数据；
- 陀螺仪 Z 轴零偏校准。

初始化顺序：

1. 上电等待至少 11 ms；
2. 读取 `WHO_AM_I`，期望值为 `0xEA`；
3. 根据 AD0 确认地址 `0x68` 或 `0x69`；
4. 写 `PWR_MGMT_1` 清除睡眠并选择 `CLKSEL=1`；
5. 使能加速度计和陀螺仪；
6. 静止采样计算陀螺仪 Z 轴零偏。

第一阶段建议以 100 Hz 轮询读取，不立即引入 DMP 和磁力计。磁力计容易受到电机磁场影响，后续若使用必须进行硬铁、软铁校准。

## 3. 软件三层结构

每个模块同时提供 `.h` 和 `.c` 文件。头文件只放外部可见的类型、宏、常量、函数声明和必要的库包含；具体代码放在 `.c` 文件中。生成的 SysConfig 文件不纳入手工业务代码。

### 3.1 BSP 硬件抽象层

```text
bsp/
  board_config.h
  tb6612.h
  tb6612.c
  encoder.h
  encoder.c
  track.h
  track.c
  icm20948.h
  icm20948.c
  scheduler.h
  scheduler.c
```

职责：

- `board_config.h`：轮径、轮距、编码器分辨率、PWM 周期、方向反相宏；
- `tb6612.c`：方向输出、PWM 占空比、停止和刹车；
- `encoder.c`：TIMG0/TIMG6 捕获 ISR、Q 方向读取、累计计数、速度采样；
- `track.c`：五路 GPIO 读取、输入 bit mask、黑白极性处理；
- `icm20948.c`：I2C 寄存器读写、初始化、原始数据读取、零偏校准；
- `scheduler.c`：TIMG7 ZERO ISR，只维护节拍计数和任务标志。

建议使用生成头文件中的实例宏，例如 `ENCODER1_INST`、`ENCODER1_INST_INT_IRQN`、`TRACK3_OUT1_PIN`，不要重新硬编码外设地址。

### 3.2 控制算法层

```text
control/
  pid.h
  pid.c
  wheel_speed.h
  wheel_speed.c
  line_follow.h
  line_follow.c
  heading.h
  heading.c
```

职责：

- `pid.c`：通用位置式或增量式 PID、输出限幅、积分限幅、抗积分饱和；
- `wheel_speed.c`：左右轮速度 PI 环；
- `line_follow.c`：循迹位置误差、丢线、交叉线、基础速度和差速修正；
- `heading.c`：编码器差速航向、陀螺仪角速度融合、定角转弯。

### 3.3 应用状态层

```text
app/
  robot_state.h
  robot_state.c
  main.c
```

应用层负责：

- 初始化顺序；
- `INIT -> SELF_TEST -> READY -> RUN` 状态转换；
- `RUN -> LOST_LINE -> STOP`；
- 任意状态进入 `FAULT -> STOP`；
- 处理按键、蓝牙或串口命令；
- 选择手动、循迹、定角转弯和测试模式。

## 4. 控制算法设计

### 4.1 时间结构

TIMG7 产生 10 ms ZERO 事件，应用中需要：

```c
NVIC_EnableIRQ(TIMER_0_INST_INT_IRQN);
DL_TimerG_startCounter(TIMER_0_INST);
```

中断中只置 `control_tick_pending`。主循环看到标志后依次执行：

1. 读取并锁存左右编码器计数；
2. 计算左右轮速度；
3. 读取循迹 bit mask；
4. 更新循迹控制器；
5. 更新左右轮速度 PI；
6. 写入左右 PWM。

### 4.2 编码器位置与速度

若 13 PPR 是电机轴每通道参数，减速比为 1:20，则输出轴每转约有 260 个 A 相上升沿。48 mm 轮径的理论周长为：

```text
C = pi * 48 = 150.8 mm
```

理论上每个 A 相计数约对应 0.58 mm，但最终必须通过手动转轮一整圈校准。

10 ms 采样时：

```text
delta_count = count_now - count_last
speed = delta_count / counts_per_wheel_rev / 0.01 s
```

速度环使用有符号速度，左右电机方向的正负号通过实测统一。共享 ISR 与主循环的数据使用 `volatile`，读取多字节计数时使用临界区或原子快照。

### 4.3 循迹位置误差

设黑线传感器值为 1，白底为 0：

```text
sum = s1+s2+s3+s4+s5
error = (43*s1 + 15*s2 + 0*s3 - 15*s4 - 43*s5) / sum
```

规则：

- `sum == 0`：进入丢线计时；
- 全白持续 20~50 ms：停止；
- `sum == 5`：作为交叉线或宽线状态，不直接当作普通误差；
- 只有一个或多个有效传感器时：使用加权误差；
- 短暂丢线：使用上一次有效误差进行减速搜索。

### 4.4 PID 环路

建议实现两个并行的轮速 PI 环和一个循迹 PD 环：

```text
循迹 PD：传感器位置误差 -> 左右轮目标速度差
左右轮 PI：目标轮速 -> 实际轮速 -> 左右 PWM
```

循迹环计算：

```text
correction = Kp_line * error + Kd_line * (error - error_last) / dt
right_target = base_speed - correction
left_target  = base_speed + correction
```

轮速环分别对左右轮执行 PI：

```text
u = Kp_speed * speed_error + integral
integral += Ki_speed * speed_error * dt
```

必须实现：

- 输出限幅；
- 积分限幅；
- 输出饱和时停止继续积分；
- 电机静止或丢线时清除积分。

ICM-20948 航向环不与循迹环强行叠加。直线循迹时可使用小比例航向修正；定角转弯时使用独立的角度/角速度控制，将其输出转换为左右轮相反速度。

### 4.5 角度与转弯

编码器差速可以估计航向变化：

```text
delta_yaw = (right_distance - left_distance) / wheel_track
```

陀螺仪 Z 轴可用于短时间转弯动态响应。建议采用编码器角度作为长期约束、陀螺仪角速度作为短期变化的互补滤波，不在第一版使用完整 DMP。

## 5. 故障处理

- ICM 初始化失败或 WHO_AM_I 错误：记录故障并禁止自动运行；
- 编码器长时间无脉冲：标记轮子失速，停止电机；
- 循迹全白超时：停止并保持方向安全；
- 传感器输入出现不可能组合：降低速度并进入诊断状态；
- PWM、方向和状态机初始化期间：强制占空比 0；
- 任何故障状态：不依赖 STBY，直接设置 PWM 0。

## 6. 验证顺序

1. 生成 SysConfig，确认 `TRACK3` 五路均为输入、PWM 比较值为 0、TIMG0/TIMG6 生成 `CC0_DN` 中断、TIMG7 生成 ZERO 中断。
2. 编译工程，确认没有 SysConfig 错误、链接错误或未定义 IRQ。
3. 不接电机，验证四个方向 GPIO 的逻辑组合。
4. 只接 TB6612，验证 0%、低速、中速、正反转和停止；确认两路电机方向与软件符号一致。
5. 逐个覆盖循迹传感器，记录黑白电平和所有 bit mask。
6. 用示波器观察 A 相、B 相、74HC74 Q 和 MSPM0 捕获中断，确认每个 A 上升沿只产生一次计数。
7. 手动转轮一整圈，校准输出轴每圈计数和方向符号。
8. 开环测量左右轮速度，记录 PWM 与速度关系。
9. 启用左右轮 PI，测试不同目标速度的稳态误差和响应时间。
10. 启用循迹 PD，先低速直线，再测试弯道、丢线和交叉线。
11. 最后加入 ICM 初始化、零偏校准和定角转弯。

## 7. 项目实现步骤

1. 保持当前 SysConfig 配置，建立 `bsp/`、`control/`、`app/` 目录。
2. 先实现 TB6612 驱动和电机极性测试。
3. 实现编码器捕获 ISR、位置计数和速度计算。
4. 实现循迹输入读取与 bit mask 诊断输出。
5. 实现通用 PID、左右轮 PI 和低速参数标定。
6. 实现循迹 PD 和丢线/交叉线状态机。
7. 实现 ICM-20948 最小驱动和角速度读取。
8. 实现定角转弯和编码器/陀螺仪互补航向。
9. 增加串口或蓝牙调参接口，至少支持基础速度、Kp、Kd、轮速 PI 参数和丢线延时修改。
10. 每完成一个模块都保留独立测试模式，比赛时只替换应用状态机，不重写底层驱动。

## 8. 当前验证结果

已完成：

- `mspm0-ccs` 静态 SysConfig 检查通过；
- SysConfig 重新生成通过；
- `ENCODER1/2` 生成 `DL_TIMERG_INTERRUPT_CC0_DN_EVENT`；
- `TIMER_0` 生成 `DL_TIMERG_INTERRUPT_ZERO_EVENT`；
- 两路 TB PWM 初始比较值为 0；
- 使用 `mingw32-make -C Debug all` 完整编译和链接通过。

SysConfig 仍会输出 ADC 自动掉电采样提示和部分 STOP/STANDBY 外设保持提示。这些是信息级提示，不是当前编译错误；本项目暂不使用低功耗模式，后续进入低功耗时再处理外设保存/恢复。

当前只完成源码、SysConfig 和编译级验证，尚未连接实际小车进行电机、编码器、循迹和 ICM 的硬件验证。
