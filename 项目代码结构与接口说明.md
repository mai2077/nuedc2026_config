# MSPM0G3507 电赛小车项目代码结构与接口说明

本文档依据 2026-07-23 工作区中的实际源码整理，覆盖 `bsp/`、`control/`、`app/` 三个目录。文中“当前启用”指当前 `app/main.c` 主流程确实会调用；“条件启用”指只有选择调试模式或打开编译宏时调用；“保留未接入”指源码仍在，但当前主流程没有调用。

## 1. 文件组织约定

- `.h` 文件对外提供类型、宏、函数声明以及调用者需要包含的库。
- `.c` 文件保存具体实现、模块私有状态、私有辅助函数和硬件操作代码。
- 调用其他模块时应包含对应 `.h`，不要直接包含 `.c`。
- `empty.syscfg` 是引脚、外设、时钟和中断配置的源文件。
- `Debug/ti_msp_dl_config.c/.h` 等文件由 SysConfig/CCS 生成，只用于查看生成名称和结果，不应手工编辑。
- `bsp` 可以依赖 TI DriverLib 和 `ti_msp_dl_config.h`；`control` 原则上只处理算法和数值；`app` 负责把硬件输入、算法和运行状态组织起来。

## 2. 三层关系

```text
app：启动、调度、按键逻辑、调试命令、运行状态
  |
  +--> control：循迹外环、轮速 PI 内环、角度控制
  |
  +--> bsp：电机、编码器、按键、循迹、UART、OLED、IMU
          |
          +--> SysConfig 生成宏 + TI DriverLib + 实际硬件
```

推荐依赖方向是 `app -> control -> bsp`。当前少数控制模块只依赖另一个控制模块中的公共常量，例如 `line_follow.c` 使用 `wheel_speed.h` 的最大速度限制；控制层本身不直接读 GPIO 或写 PWM。

## 3. 当前实际运行流程

### 3.1 启动流程

`main()` 按以下顺序启动：

1. `SYSCFG_DL_init()` 初始化 SysConfig 配置的时钟、GPIO 和外设。
2. `DEBUG_UART_initRxInterrupt()` 启用 RESV1 UART 接收中断。
3. `ENCODER_init()` 清零计数并启用两个编码器捕获中断。
4. `PI_TUNER_init()` 初始化按键、循迹控制、角度控制和轮速控制，并保持电机停止。
5. `OLED_Init()` 显示等待 KEY1 和当前 `curveBias` 的页面。
6. 启用并启动 10 ms 周期定时器 `TIMER_0`。

### 3.2 10 ms 控制链

```text
TIMER_0_INST_IRQHandler
  -> ENCODER_updateSpeed10ms()
  -> gTick10ms++

main 检测到新 tick
  -> PI_TUNER_update10ms()
     -> KEY_scan10ms()
     -> TRACK_readRawMask()
     -> LINE_FOLLOW_update10ms()
     -> WHEEL_SPEED_setTargetsMmps()
     -> WHEEL_SPEED_update10ms()
     -> TB6612_setMotor1()/setMotor2()
```

中断中没有 PID、循迹或转弯计算，只更新编码器 10 ms 差值并递增节拍。主循环使用 `now != lastControlTick` 执行一次控制；如果主循环阻塞并跨过多个 tick，不会逐个补算丢失的周期。

### 3.3 当前人机操作和启用状态

- 上电后电机不转，`curveBias` 默认为 `0 mm/s`。
- 启动前 KEY3 每次增加 `5 mm/s`，KEY4 每次减少 `5 mm/s`，范围为 `-500..+500 mm/s`。
- KEY1 经两次 10 ms 稳定采样后启动，启动事件所在周期只切换状态，下一控制周期开始输出目标速度，因此软件附加延迟最多约 10 ms。
- KEY1 启动后 `curveBias` 锁定；KEY2 在当前主流程中没有停止功能。
- 基础速度当前固定为 `500 mm/s`。
- RESV1 UART 命令和调试输出启用。
- `PI_TUNER_ENABLE_BT_COMMANDS` 和 `PI_TUNER_ENABLE_BT_TELEMETRY` 默认均为 `0`，而且 `main()` 没有调用 `BT_UART_initRxInterrupt()`，所以蓝牙路径当前未启用。
- 正常循迹不会初始化 IMU；只有通过 RESV1 UART 选择 IMU 调试模式时才按需初始化。
- 方形轨迹转弯、角度控制和转弯声光提示源码保留，但当前循迹主流程已断开这些调用。

### 4.2 `board_config.c/.h`

`board_config.h` 定义编码器方向锁存电平的极性：

- `ENCODER1_DIR_HIGH_IS_POSITIVE = 0`：右轮 DIR1 低电平记为正计数。
- `ENCODER2_DIR_HIGH_IS_POSITIVE = 1`：左轮 DIR2 高电平记为正计数。

`board_config.c` 当前只是空实现占位，没有函数。修改机械方向或 D 触发器 Q 极性时，应优先修改这两个宏，而不是改捕获 ISR 的公共逻辑。

### 4.3 `tb6612.c/.h`

关键宏：`TB6612_PWM_MAX=1000` 是软件满量程，`TB6612_TEST_DUTY=150` 是极性测试占空命令。正值表示软件定义的正转，负值表示反转。

| 接口 | 作用 |
| --- | --- |
| `TB6612_init()` | 调用 `TB6612_stopAll()`，将两个通道置于安全停止状态。 |
| `TB6612_setMotor1(signedPwm)` | 设置电机 1，输入限制到 `-1000..1000`，符号决定 AIN1/AIN2，绝对值决定 CC0 PWM。 |
| `TB6612_setMotor2(signedPwm)` | 设置电机 2，行为相同，使用 BIN1/BIN2 和 CC1 PWM。 |
| `TB6612_stopAll()` | 两路 PWM 置为 0% 等效比较值，四个方向脚全部拉低，并清除上次方向记录。 |

当命令方向改变时，驱动会先把 PWM 置零、两个方向脚拉低，等待一个 PWM 周期后再设置新方向，避免瞬间反接。当前右轮电机 1 物理反装，因此应用层最终输出使用 `-WHEEL_SPEED_getRightPwm()`；BSP 本身保持统一的符号定义。

### 4.4 `encoder.c/.h`

| 接口 | 作用 |
| --- | --- |
| `ENCODER_init()` | 清零位置/速度，清除并使能 ENCODER1、ENCODER2 捕获 IRQ。 |
| `ENCODER_reset()` | 将左右累计位置、上次位置和 10 ms 计数差全部清零。 |
| `ENCODER_updateSpeed10ms()` | 对当前累计位置与上次快照做差，得到每 10 ms 的有符号脉冲数。应由 10 ms 节拍调用。 |
| `ENCODER_getRightPosition()` | 返回右轮累计有符号位置计数。 |
| `ENCODER_getLeftPosition()` | 返回左轮累计有符号位置计数。 |
| `ENCODER_getRightSpeed10ms()` | 返回右轮最近 10 ms 的有符号计数差。 |
| `ENCODER_getLeftSpeed10ms()` | 返回左轮最近 10 ms 的有符号计数差。 |
| `ENCODER1_INST_IRQHandler()` | 右轮 TIMA0 捕获 ISR；每个 CC0 down 事件读取 DIR1 并按配置极性加一或减一。 |
| `ENCODER2_INST_IRQHandler()` | 左轮 TIMG 捕获 ISR；读取 DIR2 并按配置极性加一或减一。 |

内部位置变量使用 `uint32_t`，减计数依靠模 2^32 回绕；getter 和位置差转换成 `int32_t` 后得到预期的有符号结果。这要求单次读取范围不要跨过有符号 32 位可表达范围。

### 4.5 `track.c/.h`

bit 定义为：bit0=OUT1、bit1=OUT2、bit2=OUT3、bit3=OUT4、bit4=OUT5。当前硬件约定是黑线输出高电平，因此 mask 中的 `1` 表示对应探头检测到黑线。

| 接口 | 作用 |
| --- | --- |
| `TRACK_packRawPins(gpioPins)` | 把 DriverLib 返回的实际 GPIO pin 位映射成连续的低 5 位，便于算法和主机测试。 |
| `TRACK_readRawMask()` | 一次读取五个 TRACK3 GPIO，并返回 `0x00..0x1F` 的 mask。 |

`TRACK_MASK_OUT1..OUT5` 用于单独测试某个探头，`TRACK_MASK_ALL=0x1F` 用于过滤和判断全黑。

### 4.6 `key.c/.h`

按键 GPIO 上拉，按下接地，所以接口统一输出“按下为 1”的逻辑 mask。

| 接口 | 作用 |
| --- | --- |
| `KEY_init()` | 清空候选值、稳定值和连续采样计数。 |
| `KEY_readPressedMask()` | 立即读取原始 GPIO，转换成 `KEY_MASK_KEY1..KEY4`，不消抖。 |
| `KEY_scan10ms(stableMask)` | 每 10 ms 调用；相同候选值连续出现 2 次且与旧稳定值不同才返回 `true`，并写出新的稳定 mask。空指针返回 `false`。 |

该接口既会上报按下变化，也会上报释放变化；应用层需要检查返回的 mask 中具体哪个按键位为 1。

### 4.7 `debug_uart.c/.h`

RESV1 UART 的发送函数使用阻塞式 DriverLib 发送；接收使用 128 字节环形缓冲和 RX/RX timeout/overrun 中断。缓冲满时丢弃最旧字节，没有公开溢出计数。

| 接口 | 作用 |
| --- | --- |
| `DEBUG_UART_writeByte(data)` | 阻塞发送一个字节。 |
| `DEBUG_UART_writeString(text)` | 发送以 `\0` 结束的字符串。 |
| `DEBUG_UART_writeUInt32(value)` | 以十进制发送无符号 32 位整数。 |
| `DEBUG_UART_writeInt32(value)` | 以十进制发送有符号 32 位整数，正确处理 `INT32_MIN`。 |
| `DEBUG_UART_writeHexByte(value)` | 发送两位大写十六进制。 |
| `DEBUG_UART_writeSignedFixed3(milliValue)` | 把千分单位整数显示为带 3 位小数的有符号数。 |
| `DEBUG_UART_writeTrackFrame(rawMask)` | 输出 `TRACK mask=0x.. bits=.....\r\n` 诊断文本。 |
| `DEBUG_UART_initRxInterrupt()` | 清空软件缓冲，配置一级 RX FIFO 阈值并使能 UART/NVIC 接收中断。 |
| `DEBUG_UART_tryReadByte(data)` | 非阻塞取一个接收字节；成功返回 1，无数据或空指针返回 0。 |
| `RESV1_UART_INST_IRQHandler()` | 将硬件 RX FIFO 数据搬入软件环形缓冲。 |

### 4.8 `bt_uart.c/.h`

蓝牙 UART 提供 1024 字节 RX 环形缓冲和 512 字节 TX 环形缓冲，收发均由中断配合 FIFO 完成。RX 满时覆盖最旧数据并累计饱和到 65535 的溢出计数；TX 满时同样覆盖最旧待发数据，但没有公开 TX 溢出计数。

| 接口 | 作用 |
| --- | --- |
| `BT_UART_writeByte(data)` | 临时屏蔽本 UART IRQ，把字节放入 TX 环并启动填充 FIFO；调用方不等待实际发送完成。 |
| `BT_UART_writeString(text)` | 非阻塞队列化字符串。 |
| `BT_UART_writeUInt32(value)` | 队列化十进制无符号整数。 |
| `BT_UART_writeInt32(value)` | 队列化十进制有符号整数。 |
| `BT_UART_writeSignedFixed3(milliValue)` | 队列化带 3 位小数的定点文本。 |
| `BT_UART_writeTrackFrame(rawMask)` | 队列化循迹 mask 诊断文本。 |
| `BT_UART_initRxInterrupt()` | 清空 RX/TX 环和溢出计数，配置 FIFO 阈值、UART 中断和最高 NVIC 优先级。 |
| `BT_UART_tryReadByte(data)` | 非阻塞读取一个字节。 |
| `BT_UART_getAndClearRxOverflowCount()` | 原子读取并清零 RX 溢出次数。 |
| `BT_UART_INST_IRQHandler()` | 循环处理所有挂起的 RX、TX、超时和溢出事件，并搬运 FIFO。 |

当前 `main()` 未初始化该模块，`PI_TUNER` 的两个蓝牙编译宏也默认为 0，因此这些接口只是保留能力。

### 4.9 `bsp_iic.c/.h`

`IIC_Status_t` 包含 `IIC_OK`、`IIC_ERR`、`IIC_BUSY`、`IIC_TIMEOUT`。对外兼容接口使用移位后的 8 位设备地址，例如 ICM-20948 的 `0x68 << 1`；实现内部会右移成 DriverLib 所需的 7 位地址。

全局对象 `UserII2Dev` 是 `IICInterface_t`，各函数指针作用如下：

| 接口成员 | 作用 |
| --- | --- |
| `write(address, data, size, timeout)` | 向设备连续写字节并产生 STOP。 |
| `read(address, data, size, timeout)` | 从设备连续读取指定长度并产生 STOP。 |
| `write_reg(address, reg, data, size, timeout)` | 先发送 8 位寄存器地址再发送数据；单次数据最多 7 字节。 |
| `read_reg(address, reg, data, size, timeout)` | 无 STOP 写寄存器地址，再 repeated-start 读数据。 |
| `delay_ms(ms)` | 使用 CPU 周期忙等待毫秒延时。 |

轮询过程检查空闲、BUSY 和 ERROR，超时或错误时强制 STOP、复位传输、清空 TX/RX FIFO 并清中断标志。`timeout=0` 会按至少 1 ms 处理。

### 4.10 `bsp_imu.c/.h` 与寄存器头文件

`bsp_imu` 同时保留 ICM-20948 和 MPU6500/9250 风格设备接口。主要数据类型：

- `PrivateBuf_t`：三轴 `x/y/z` 浮点值。
- `IMU_DATA_t`：陀螺仪 rad/s、加速度 m/s²、磁场 uT。
- `ATTITUDE_DATA_t`：roll/pitch/yaw，单位为弧度。
- `IMUDevice_t`：选择 ICM-20948 或 MPU6500 路径。
- `bsp_imu_diag_t`：记录初始化阶段、地址、WHO_AM_I、磁力计 ID/可用性和最后 I2C 状态。

| 普通接口 | 作用 |
| --- | --- |
| `bsp_imu_select_device(device)` | 选择后续统一接口使用的 IMU；非法枚举会被忽略。 |
| `bsp_imu_get_selected_device()` | 返回当前选择。 |
| `bsp_imu_get_device_name(device)` | 返回 `ICM20948`、`MPU6500` 或 `UNKNOWN`。 |
| `bsp_imu_get_interface()` | 返回 `UserICM20948` 或 `UserMPU9250` 接口对象指针。 |
| `bsp_imu_get_diag(diag)` | 复制当前设备的初始化诊断信息；空指针不操作。 |
| `bsp_imu_update_9axis_checked(data)` | 执行一次带错误返回值的九轴读取，成功返回 0。 |
| `bsp_imu_port_get_tick_ms()` | 姿态算法的弱时间钩子；默认返回 0，此时固定使用 20 ms 步长，平台可提供同名强定义覆盖。 |

全局 `IMUInterface_t` 对象 `UserICM20948` 和 `UserMPU9250` 均包含以下兼容函数指针：

| 接口成员 | 作用 |
| --- | --- |
| `Init()` | 检测 WHO_AM_I、复位并配置加速度计/陀螺仪和磁力计；成功返回 0。 |
| `DeInit()` | 令设备进入睡眠并清除运行状态。 |
| `UpdateZeroPoint_axis(point)` | 保存三轴零偏，并令低通滤波器从下一样本重新起步。 |
| `UpdateZeroPoint_attitude(attitude)` | 保存姿态零点，后续输出减去该零点。 |
| `Update_9axisVal(data)` | 读取九轴数据，但兼容接口不向调用者返回错误码。需要错误判断时使用 checked 接口。 |
| `UpdateAttitude(data, attitude)` | 用加速度校正 roll/pitch、陀螺仪积分四元数并输出姿态；当前不使用磁力计修正 yaw。 |

读取过程包含加速度、陀螺仪和磁力计低通滤波，Z 轴陀螺仪有 `0.05 rad/s` 死区。`icm_20948_reg.h` 和 `mpu9250_reg.h` 仅定义寄存器/设备 ID，没有可调用函数。

### 4.11 `icm20948.c/.h`

这是建立在 `bsp_iic`/`bsp_imu` 上的最小兼容驱动。`ICM20948_Status` 区分参数错误、超时、I2C 错误、WHO_AM_I 不符和配置不符；`ICM20948_GyroSample` 同时保存三轴 raw 与 mdps。

| 接口 | 作用 |
| --- | --- |
| `ICM20948_init(whoAmI)` | 选择 ICM-20948 并调用统一驱动初始化，返回状态并写回 WHO_AM_I。 |
| `ICM20948_selfCheck(whoAmI)` | 回读 WHO_AM_I、PWR_MGMT、采样分频和陀螺仪配置，确认与预期一致。 |
| `ICM20948_readGyro(sample)` | 从 GYRO_XOUT_H 起批量读取三轴原始值，并计算 mdps。 |
| `ICM20948_rawToMilliDps(rawValue)` | 按 ±2000 dps 量程把 raw 转换为 mdps。 |

### 4.12 `ano_protocol.c/.h`

| 接口 | 作用 |
| --- | --- |
| `ANO_sendImuFrame(sample)` | 通过 RESV1 UART 发送功能 ID `0x01` 的加速度/陀螺仪二进制帧，附和校验与累加校验。 |
| `ANO_sendEulerFrame(attitude, fusionStatus)` | 发送功能 ID `0x03` 的 roll/pitch/yaw 厘度帧。 |
| `ANO_accelToCmss(value)` | m/s² 转 cm/s²，四舍五入并饱和到 `int16_t`。 |
| `ANO_gyroToCounts(value)` | rad/s 转 dps 后按 16.384 count/(deg/s) 缩放并饱和。 |

空样本指针不会发送。当前 IMU 调试模式实际调用的是欧拉角帧。

### 4.13 `oled_soft_i2c.c/.h`、`oled.c/.h`、`oled_font.h`

软件 I2C 以约 100 kHz 运行，通过“关闭输出”释放 SCL/SDA，依赖板上外部上拉。发送第九个时钟但不采样 ACK，因此接口不能检测 OLED 缺失或总线 NACK。

| 软件 I2C 接口 | 作用 |
| --- | --- |

## 5. Control 层

控制层输入测量值和目标值，输出左右轮速度或 PWM 命令。当前完整控制结构是：循迹位置 PID 外环产生左右轮速度目标，左右轮各自的 PI 内环产生 PWM。

### 5.1 控制文件总览

| 文件 | 作用 | 当前状态 |
| --- | --- | --- |
| `line_follow.c/.h` | 五路循迹加权位置误差、PID 外环、丢线状态 | 当前启用 |
| `wheel_speed.c/.h` | 左右轮独立 PI、前馈、限幅和编码器单位换算 | 当前启用 |
| `angle_control.c/.h` | 原地定角转向、停稳、前进和航向 PID 状态机 | 保留未接入 |

### 5.2 `wheel_speed.c/.h`：轮速内环

每个车轮有独立 `WheelSpeedPi` 私有控制器。公共诊断结构 `WHEEL_SPEED_Diagnostics` 提供目标速度、实测速度、PWM、误差、P/I/前馈/D 分量和增益。

关键参数：

- 编码器：`265 count/rev`。
- 轮径：`48 mm`，周长以 `150796/1000 mm` 定点保存。
- 目标限幅：`±1500 mm/s`。
- PWM 限幅：`±700`，未使用 TB6612 的全部 1000 范围。
- 非零目标固定加入 `±200 PWM` 前馈。
- 当前左右轮默认 `Kp=20.00`、`Ki=0.00`，D 始终为 0。

| 接口 | 作用 |
| --- | --- |
| `WHEEL_SPEED_init()` | 两轮目标和状态清零，载入默认 PI。 |
| `WHEEL_SPEED_setTargetsMmps(right, left)` | 设置并限幅两轮目标；目标为 0 或跨越正负方向时重置 PI 累积。 |
| `WHEEL_SPEED_update10ms(rightCounts, leftCounts)` | 用最近 10 ms 编码器计数分别更新两轮 PI 输出。 |
| `WHEEL_SPEED_setGainsX100(wheel, kpX100, kiX100)` | 设置指定轮增益；P 最大 100.00、I 最大 20.00，成功返回 1；变化时清除控制器动态项。 |
| `WHEEL_SPEED_getDiagnostics(wheel, diagnostics)` | 复制指定轮的完整诊断快照。 |
| `WHEEL_SPEED_getRightTargetMmps()` | 返回右轮目标速度。 |
| `WHEEL_SPEED_getLeftTargetMmps()` | 返回左轮目标速度。 |
| `WHEEL_SPEED_getRightPwm()` | 返回右轮软件方向下的有符号 PWM。 |
| `WHEEL_SPEED_getLeftPwm()` | 返回左轮有符号 PWM。 |
| `WHEEL_SPEED_mmpsToCountsQ8(speed)` | 将 mm/s 转成每 10 ms 计数的 Q8 定点值，供 PI 减少量化误差。 |
| `WHEEL_SPEED_counts10msToMmps(counts)` | 将 10 ms 有符号计数换算成 mm/s。 |

控制式可概括为：

```text
error = target_counts_Q8 - measured_counts * 256
output = signed_feedforward + Kp * error + integral
```

积分候选先限制在 `±700`；输出饱和时，只有误差方向能把输出拉回非饱和区才接受该积分候选，形成条件积分抗饱和。结构中保留 `derivativePwm` 诊断字段，但当前始终为 0。

### 5.3 `line_follow.c/.h`：循迹位置外环

五个传感器位置按 OUT1 到 OUT5 定义为 `{+43,+15,0,-15,-43} mm`。检测到多个黑线点时，误差是所有有效位置的算术平均值。

当前参数：`Kp=5.0`、`Ki=0`、`Kd=0.08`，修正上限为 `min(baseSpeed,150 mm/s)`，积分状态限幅 `±200 mm·s`，全白停止阈值为 15 个 10 ms 周期，即约 150 ms。

速度分配为：

```text
right = base + curveBias - correction
left  = base - curveBias + correction
```

输出目标最终限制在 `0..1500 mm/s`，因此当前循迹外环不命令车轮反转。

| 接口 | 作用 |
| --- | --- |
| `LINE_FOLLOW_init()` | 载入默认 PID、清状态和历史误差，初始输出停止。 |
| `LINE_FOLLOW_setBaseSpeedMmps(speed)` | 设置基础速度并限制在 `0..1500 mm/s`。 |
| `LINE_FOLLOW_setCurveBiasMmps(bias)` | 设置固定差速；发生变化时清除循迹动态状态。该底层接口本身不限幅。 |
| `LINE_FOLLOW_pidIsValid(kp,ki,kd)` | 检查非 NaN、非负及 P≤50、I≤20、D≤5。 |
| `LINE_FOLLOW_setPid(kp,ki,kd)` | 校验并更新 PID；成功返回 1，参数变化时清积分和微分历史。 |
| `LINE_FOLLOW_resetDynamics()` | 清除最后有效误差、丢线计数、积分和微分历史，并输出停止。 |
| `LINE_FOLLOW_update10ms(rawMask, output)` | 执行一次状态判断和 PID，保存并可选写出结果。 |
| `LINE_FOLLOW_getOutput(output)` | 复制最近一次 `LINE_FOLLOW_Output`。 |

`LINE_FOLLOW_State` 的实际行为：

- `TRACKING`：存在 1~4 个黑线传感器，按加权位置误差运行 PID。
- `INTERSECTION`：五路全黑，清 PID 历史，误差视为 0，按 base 和 bias 直行。
- `HOLD_LAST`：短暂全白且曾有有效误差，继续使用最后误差搜索；首次上电没有有效误差时直接停止。
- `LOST`：全白达到 15 tick 后左右目标归零。

`LINE_FOLLOW_Output` 包含原始/有效 mask、状态、丢线计数、PID 参数、误差、积分/微分状态、P/I/D 修正分量、总修正、差速和左右速度目标，适合串口调试。

### 5.4 `angle_control.c/.h`：角度/航向控制

该模块当前保留未接入。它的状态顺序是：

```text
IDLE
 -> TURN_OPEN_LOOP（左右轮 ±220 mm/s 原地转）
 -> TURN_HOLD（目标附近停止 50 tick）
 -> FORWARD_FIXED（固定前进 50 tick）
 -> DRIVE_PID（航向 PID 前进）
```

角度误差包装到 `[-180,180]`。原地转弯在误差进入 `±5°` 或检测到越过目标时结束；前进 PID 修正限制为 `min(abs(baseSpeed),200 mm/s)`。

| 接口 | 作用 |
| --- | --- |
| `ANGLE_CONTROL_init()` | 初始化为 IDLE，PID、目标和输出清零。 |
| `ANGLE_CONTROL_disable()` | 立即回到 IDLE、清历史并令输出为零。 |
| `ANGLE_CONTROL_isActive()` | 非 IDLE 返回 1。 |
| `ANGLE_CONTROL_setBaseSpeedMmps(speed)` | 设置转弯后固定前进和航向保持的基础速度；本接口不额外限幅。 |
| `ANGLE_CONTROL_setPid(kp,ki,kd)` | 分别接受非 NaN、非负增益，发生变化时清 PID 历史。 |
| `ANGLE_CONTROL_setTargetYawDeg(target)` | 归一化目标角并进入开环转弯状态。 |
| `ANGLE_CONTROL_wrapErrorDeg(target,current)` | 返回包装到 `[-180,180]` 的最短角度误差。 |
| `ANGLE_CONTROL_update10ms(currentYaw,output)` | 推进状态机并生成左右轮速度目标。 |
| `ANGLE_CONTROL_getOutput(output)` | 复制最近输出。 |
| `ANGLE_CONTROL_stateName(state)` | 返回便于串口显示的状态字符串。 |

`ANGLE_CONTROL_Output` 提供左右目标、状态、当前/目标/误差角度及转向修正。方形导航源码在角度控制进入 `TURN_HOLD` 的首个周期就会禁用它并切回循迹，因此独立模块中的后续 500 ms 停稳和固定前进阶段在该组合路径下不会继续执行。

尽管文件名仍叫 PI tuner，它现在承担主运行编排。固定目标速度为 `500 mm/s`，每 10 ms 读取循迹、运行位置外环，再运行两个轮速 PI 内环，最后对物理反装的右轮电机 1 取反输出。

| 公共接口 | 作用 |
| --- | --- |
| `PI_TUNER_init()` | 初始化 wheel/angle/line/key，设置基础速度与默认差速，停止电机，清命令/调试/IMU状态，并打印 RESV1 调试菜单。 |

RESV1 文本 PI 命令格式为：

```text
SET P:20.00 I:0.00 D:0.00\n
```

P/I 会同时应用到左右轮；D 会被解析和显示，但轮速控制器没有 D 项，因此被忽略。支持最多 6 位小数输入并四舍五入到百分之一。调试选择为 `1/mode1`（50 ms PWM 二进制帧）、`2/mode2`（20 ms IMU 欧拉角帧）、`3/mode3`（500 ms PID 文本），`0/menu/m` 返回菜单。只在 mode2 中按需初始化 IMU。

蓝牙路径支持文本 SET 命令和 `0xA5 ... checksum 0x5A` 二进制参数包，但 `PI_TUNER_ENABLE_BT_COMMANDS` 默认为 0。二进制包可携带轮速 PI、角度 PID、循迹 PID 和目标 yaw；当前即使启用，包也只在小车已经运行时应用。

## 7. 当前闭环结构总结

当前真正运行的是两层串级控制：

```text
五路循迹位置
  -> 位置误差（mm）
  -> 循迹 PID 外环
  -> 左右目标速度（mm/s）
  -> 左右编码器速度反馈
  -> 两个独立 PI + 前馈内环
  -> 左右有符号 PWM
  -> TB6612 和电机
```

- 外环 P 决定偏离黑线时的主要转向强度。
- 外环 D 抑制横向误差快速变化，降低摆动；I 当前为 0，避免传感器离散误差积累。
- `curveBias` 在 PID 修正之外预置固定左右差速，适合持续弯道实验。
- 两个速度内环分别补偿左右电机差异；当前只有 P 和固定前馈，I 默认为 0。
- 角度 PID 是另一条可选外环，不与当前循迹外环同时工作；方形转弯代码目前也未接回主链。

## 8. 修改和扩展时的边界

1. 换引脚、外设实例、定时器模式或中断配置：修改 `empty.syscfg` 并重新生成。
2. 换编码器方向判定电平：优先修改 `board_config.h` 的极性宏。
3. 换电机机械安装方向：在 app 的最终电机映射处统一处理，不要让控制算法感知机械反装。
4. 调轮速：修改 `wheel_speed.h` 默认值，或运行时调用 `WHEEL_SPEED_setGainsX100()`。
5. 调循迹：调用 `LINE_FOLLOW_setPid()`，并结合 `LINE_FOLLOW_Output` 观察误差与各项修正。
6. 恢复蓝牙：除打开 `PI_TUNER` 编译宏外，还要在启动阶段调用 `BT_UART_initRxInterrupt()`。
7. 恢复方形转弯：应由 app 统一初始化 IMU/提示模块，把 yaw 有效性传给 `SQUARE_NAV_update10ms()`，再将其左右目标交给轮速内环。
8. 所有 `update10ms()` 接口必须保持稳定 10 ms 调用；串口阻塞输出、OLED 全屏刷新和 I2C 长延时不应放进定时器 ISR。
